C4-діаграма — це метод візуалізації архітектури програмного забезпечення, який розробив Саймон Браун. Вона розбиває архітектуру на чотири рівні абстракції, звідси і назва:

Level 1: System Context — показує систему в цілому: як вона взаємодіє з користувачами та зовнішніми системами.
Level 2: Container — деталізує основні компоненти системи: веб-додаток, мобільний додаток, база даних, API тощо.
Level 3: Component — розкриває внутрішню структуру контейнера: які саме модулі/сервіси/класові компоненти в ньому.
Level 4: Code (не обов'язковий) — деталізує реалізацію до рівня класів (використовується рідко).

Це зручно для команд, які хочуть узгоджено документувати, обговорювати або представляти архітектуру на різних рівнях деталізації.
